# 実際のセレクタ・操作フローの記録方法

このツールは、カイポケの実際のログイン画面・操作画面を私たち（開発アシスタント）が直接見ることができないため、
あなた自身に `playwright codegen` で実際の操作を記録してもらい、その結果をコードに反映する形で作ります。

## 事前準備

```bash
npm install
npx playwright install chromium   # 初回のみ
```

## 1. ログインフローの記録

```bash
npx playwright codegen
```

- 起動したブラウザで、手動でカイポケのログインページへ移動してください。
- 法人ID・ユーザーID・パスワードを入力し、ログインしてください。
- 別ウィンドウの「Playwright Inspector」に生成されたコードが表示されます。

記録結果から、以下を `src/auth.js` に反映してください:

| 項目 | 反映先 |
|---|---|
| ログインページのURL | `LOGIN_URL` |
| 法人ID入力欄のロケータ | `SELECTORS.corpIdInput` |
| ユーザーID入力欄のロケータ | `SELECTORS.userIdInput` |
| パスワード入力欄のロケータ | `SELECTORS.passwordInput` |
| ログインボタンのロケータ | `SELECTORS.submitButton` |
| ログイン後にのみ表示される要素（例: ナビゲーションの特定項目） | `SELECTORS.loggedInIndicator` |

**ロケータの書き方の注意:** Inspectorが提示する候補のうち、可能な限り
`page.getByRole(...)` / `page.getByLabel(...)` / `page.getByText(...)` を優先してください。
生のCSSセレクタ（`page.locator(".css-xyz123")` のようなもの）はカイポケ側のUI変更で
壊れやすいため、role/labelベースのロケータが使える場合はそちらを優先します。

各値は `(page) => page.getByLabel("...")` のような、pageを受け取ってLocatorを返す関数として設定します。

もし2段階認証やSMS確認などログイン後に追加のステップがある場合は、その旨を教えてください。
現在のログインフローは単純なID/パスワードのみを想定しています。

## 2. スケジュール変更フローの記録

再度 `npx playwright codegen` を実行し、今回は以下の操作を手動で行ってください
（可能であれば本番影響のない、テスト用または後で元に戻せる対象で行ってください）:

1. 対象の利用者・訪問日を検索/絞り込みする
2. 該当する訪問予定を開く
3. 時間・サービス内容を変更する
4. 保存する
5. 保存成功が確認できる表示（トースト通知、リダイレクト先、確認ダイアログの文言など）を確認する

記録結果を `src/operations/scheduleChange.js` の `SELECTORS` に反映してください。特に以下を明確にしてください:

- 利用者名・訪問日からレコードを一意に特定する方法（検索結果が複数該当した場合にどう見分けるか）
- 保存ボタンを押す前の状態（dry-runで止める位置）
- 保存成功/失敗をコードから判定する方法

## 3. スタッフ訪問スケジュール組みフローの記録

同様に、スタッフを利用者の訪問予定へ割り当てる操作を記録し、
`src/operations/staffAssignment.js` の `SELECTORS` に反映してください。

## 4. 記録後の確認

各操作ファイルの `SELECTORS` オブジェクト内に `null` のままの項目がないか確認してください。
`null` が残っていると、実行時に「未設定です」というエラーで即座に停止するようになっています
（誤った操作を防ぐための意図的な安全策です）。
